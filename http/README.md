# 知识点

## HTTP 请求相关

1.  URL 组成部分

| 信息                  | 说明                             |
| :------------------ | :----------------------------- |
| http&#x3A;//        | 请求使用的协议                        |
| helloflask.com      | 服务器地址/域名                       |
| /hell?name=xiaoming | 要获取的资源路径（path），类似 UNIX 的文件目录结构 |

2.  请求报文：

    -   报文首部：请求行（方法、url、协议）
    -   报文首部：各种首部字段
    -   空行
    -   报文主体

3.  FLask 中 request 请求属性，获取请求相关的一系列信息

4.  Flask 中设置监听的 http 方法：  
    `@app.route('/', methods=['GET', 'POST'])`

5.  Flask 内置的 URL 变量转换器

| 转换器    | 说明             |
| :----- | :------------- |
| string | 不包含斜线的字符串（默认值） |
| int    | 整型             |
| float  | 浮点型            |
| path   | 包含斜线的字符串       |
| any    | 匹配一系列给定值中的一个元素 |
| uuid   | UUID 字符串       |

6.  Flask 请求钩子：

| 钩子                   | 说明                                            |
| :------------------- | :-------------------------------------------- |
| before_first_request | 注册一个函数，在处理第一个请求前运行                            |
| before_request       | 注册一个函数，在处理每个请求前运行                             |
| after_request        | 注册一个函数，如果没有未处理的异常抛出，会在每个请求结束后运行               |
| teardown_request     | 注册一个函数，会在每个请求结束后运行，；如果发生异常，会传入异常对象作为参数到注册的函数中 |
| after_this_request   | 在视图函数内注册一个函数，会在这个请求结束后运行                      |

7.  响应报文

    -   报文首部：响应状态行（协议、状态码、原因短语）
    -   报文首部：各种响应首部字段
    -   空行
    -   报文主体
